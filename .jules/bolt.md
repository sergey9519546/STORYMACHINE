## 2024-06-25 - React Component Re-render Performance Penalty from O(N) String Parsing
**Learning:** In highly interactive React components bound to large string state inputs (like full-screen text editors for scripts), invoking O(N) operations like `parseFountain` multiple times sequentially per render cycle acts as an unseen multiplier for keystroke latency. A single `textarea` `onChange` caused 4 full AST parses concurrently (Highlighting, Shot List, Firewall, and Statistics) causing extreme, visible stutter for large documents.
**Action:** Always extract static string-parsing logic out of the immediate component body and inner render methods, binding it to a single `useMemo` dependency hook per render. Passing this memoized AST representation downward to layout-dependent renderers avoids cumulative parsing costs.
