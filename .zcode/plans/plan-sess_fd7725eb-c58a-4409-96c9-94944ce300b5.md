## ScriptIDE Visual Redesign — same theme, professional composition

Goal: keep the paper·ink·stamp screenplay identity, but fix the flat, wireframe feel in the screenshot. No new stack, no theme change. Improve scale, hierarchy, materiality, organization, and interaction states.

### Root cause of the "flat/generic" look
1. Everything renders at one weight: 10–11px mono, hairline borders, one beige value everywhere.
2. The screenplay page floats in a dead empty canvas (huge unused left gutter).
3. Scene rail is an unstyled text list with no hierarchy or metadata.
4. Action strip + toolbar read like debug chrome, not a writing instrument.
5. No depth: no layering, shadow, or focus emphasis on the manuscript.

### Changes (scoped to the Script desk visual layer)

**1. Page stage (highest impact) — `ScriptIDE.tsx`, `FountainEditor` theme**
- Center the manuscript in a real "desk": warm graded canvas (`--sm-paper` → slightly darker edge), page gets `--sm-shadow-lg` + 1.5px ink edge so it sits like paper on a table.
- Constrain and center the page column (max ~816px) with balanced gutters instead of a left-dumped page + dead right space.
- Add restrained page furniture: top hairline + page number footer feel via existing screenplay-layout, no buffer changes.

**2. Scene navigator — `Sidebar.tsx`**
- Replace flat list rows with structured items: scene number (mono, faint), slugline (ink, medium), and INT/EXT + time chips.
- Active/hover: stamp left-edge marker (reuse `.sm-card--sel` idiom), not full-row neon.
- Section label + count header; denser but organized (VISUAL_DENSITY ~5).

**3. Production header — `Toolbar.tsx`**
- Keep night bar + Write/Coverage/Ship, but give it breathing room: title gets display-weight treatment, status becomes a proper chip cluster, export/simulate group visually separated from identity.
- Increase touch targets to 40–44px; align to a single baseline.

**4. Action strip — `ScriptIDE.tsx`**
- Reframe as a "director's slate" bar: one dominant CTA + quiet context, consistent left slug / right actions rhythm.

**5. Materiality + states (design-taste rules)**
- Tinted shadows (to background hue), inner top-border highlight on raised panels.
- Tactile `:active` (`translate-y-[1px]`/`scale-[0.98]`) on buttons; spring-free CSS transitions on the sitewide easing token.
- Real focus rings on all controls; fix the sub-44px targets on the desk.

**6. Responsive**
- Mobile: page fills width with smaller gutters; sidebar stays the existing drawer; header collapses cleanly (no clipped controls).

### Out of scope (flag, not do now)
- Deep tool panels (WhatIf, Director, etc.) migration — separate wave.
- Any workflow/logic changes from the gap audit (New Story, keyless calls, focus traps) — those are correctness waves, tracked separately.
- No theme replacement, no new fonts beyond existing Anton/Inter/JetBrains/Courier roles.

### Verification
- `npm run lint` + `npm run build` green.
- Playwright desktop (1440) + mobile (390) captures of Write / Coverage / Ship; confirm: page reads as primary, no clipped controls, scene rail hierarchy visible, targets ≥44px on coarse pointer.
