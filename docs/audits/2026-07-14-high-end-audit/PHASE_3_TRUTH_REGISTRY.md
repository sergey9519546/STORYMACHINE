**Purpose:** preserve the propositions that materially control strategy,

## Epistemic and conflict-handling rules

`Verified fact` is directly supported by code, generated artifacts, executed
witnesses, or authoritative dated text. `Strong conclusion` is the narrowest
decision-level conclusion supported by converging independent evidence streams.
`Inference` is a reasoned consequence that has not itself been observed.
`Assumption` is a premise currently being acted on without direct validation.
`Hypothesis` is a proposition designed for empirical testing. `Constraint` is a
binding project, evidence, rights, or operating boundary. `Unresolved` records
a consequential question for which the available evidence does not select one
answer.

Conflicts are resolved using this precedence for claims about the repository:
executed witness → current code/generated invariant → executable test → active
decision document → other narrative documentation → marketing copy. A later
document does not supersede a contradictory executable fact merely because it
is labelled canonical. For external claims, primary official or peer-reviewed
evidence outranks vendor pages; vendor pages establish only what the vendor
publishes. Missing observations are `unresolved`, not negative evidence.
