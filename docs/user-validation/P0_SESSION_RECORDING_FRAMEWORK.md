# P0 User Validation - Session Recording & Consent Framework

## Overview

Framework for recording, transcribing, and documenting P0 user validation sessions in compliance with consent requirements.

---

## Consent Form

### Video/Audio Recording Consent

**STORYMACHINE User Research - Recording Consent**

Thank you for participating in user research for STORYMACHINE, a screenplay analysis tool.

**Purpose of Recording:**
This session will be recorded (video and audio) for research purposes only. Recordings will be used to:
- Document user feedback and reactions
- Create written transcripts for analysis
- Capture verbatim quotes for research findings
- Inform product development decisions

**How Recordings Will Be Used:**
- ✅ Internal research and product development only
- ✅ Written transcripts and anonymized quotes in research documents
- ✅ Stored securely and deleted after project completion
- ❌ NOT shared publicly
- ❌ NOT used for marketing or promotional materials
- ❌ NOT shared with third parties

**Your Rights:**
- You may decline recording and still participate (notes only)
- You may request recording be stopped at any time
- You may request your recording be deleted after the session
- Your participation is voluntary and you may end the session at any time

**Anonymization:**
- Your name will be replaced with a pseudonym (e.g., "Participant P0-01")
- Identifying information will be redacted from transcripts
- Published quotes will be anonymous unless you explicitly grant permission

**Data Retention:**
- Recordings: Deleted within 90 days of project completion
- Transcripts: Retained with anonymized identifiers
- Research findings: Retained indefinitely with no personal identifiers

## Pre-Session Checklist

### Before Each Session:

- [ ] Send confirmation email with video link 24 hours before
- [ ] Test video recording software (OBS Studio, Zoom recording, etc.)
- [ ] Prepare consent form (digital or verbal)
- [ ] Have backup recording method ready (phone, second computer)
- [ ] Prepare session guide (questions, prompts)
- [ ] Have sample coverage report ready to share
- [ ] Set up note-taking document

### Recording Setup:

**Recommended Tools:**
- **Video:** Zoom (built-in recording), Google Meet (with extension), OBS Studio
- **Backup Audio:** Phone voice recorder, Otter.ai
- **Transcription:** Otter.ai, Rev.com, Descript, or manual transcription

**File Naming Convention:**
```
P0-[ID]-[Date]-[Pseudonym]
Example: P0-01-2026-07-20-ParticipantA.mp4
```

## Post-Session Workflow

### Immediately After Session:

1. **Save Recording:**
   - Export recording from video platform
   - Save to secure location: `recordings/P0-[NN]-[Date]-[Pseudonym].mp4`
   - Create backup copy

2. **Complete Session Notes:**
   - Fill in any gaps while memory is fresh
   - Add behavioral observations
   - Capture key quotes verbatim

3. **Send Thank You:**
   - Email participant within 24 hours
   - Thank them for their time
   - Offer free analysis if requested
   - Confirm early access list addition

### Within 48 Hours:

4. **Transcribe Recording:**
   - Use Otter.ai, Rev.com, or manual transcription
   - Save transcript: `transcripts/P0-[NN]-[Date]-[Pseudonym].txt`
   - Verify accuracy of key quotes

5. **Anonymize Transcript:**
   - Replace participant name with pseudonym
   - Redact any identifying details (company names, project titles, etc.)
   - Preserve context without revealing identity

6. **Update Recruitment Tracker:**
   - Mark session as "Completed"
   - Update participant demographics
   - Note pull signal (strong/qualified/weak/none)

### After All Sessions Complete:

7. **Synthesize Findings:**
   - Create P0_EVIDENCE_SUMMARY.md
   - Aggregate pull signals
   - Identify common themes
   - Document objections and trust requirements

8. **Make GO/NO-GO Decision:**
   - Count pull signals (need 4+ strong for GREEN)
   - Assess whether product creates demand
   - Document decision and rationale

## Security & Privacy

### Data Storage:

**Recordings:**
- Store in encrypted folder
- Access restricted to research team only
- Delete within 90 days of project completion

**Transcripts:**
- Anonymize before sharing
- Store separately from recordings
- Retain with pseudonyms only

**Research Documents:**
- Use pseudonyms only
- Aggregate data across participants
- No individual-identifying information in published findings

### Data Deletion Timeline:

- **Recordings:** Delete 90 days after P0 completion
- **Raw Transcripts:** Delete 90 days after P0 completion
- **Anonymized Transcripts:** Retain indefinitely
- **Research Findings:** Retain indefinitely (no PII)
